[정보처리기사] 1과목 소프트웨어 설계 - UI/UX 설계

Created on May 15, 2026

2026   ·   정보처리기사   소프트웨어설계   UI   UX   ·   certificate

1. UI / UX 개념

용어 설명
UI (User Interface) 사용자와 시스템 간 상호작용이 이루어지는 공간·요소
UX (User Experience) 시스템을 사용하면서 느끼는 전체적인 경험

2. UI 유형

유형 설명
CLI (Command Line Interface) 명령어 입력 방식 (터미널)
GUI (Graphical User Interface) 그래픽 기반 (Windows, macOS)
NUI (Natural User Interface) 터치, 음성, 제스처 등 자연스러운 입력 방식
VUI (Voice User Interface) 음성 기반 인터페이스 (시리, 알렉사)
OUI (Organic User Interface) 유연하고 물리적인 인터페이스 (웨어러블 등)

3. UI 설계 원칙

원칙 설명
직관성 (Intuitiveness) 별도 학습 없이 쉽게 사용 가능
유효성 (Effectiveness) 사용자 목표를 정확하게 달성 가능
학습성 (Learnability) 쉽게 배우고 기억 가능
유연성 (Flexibility) 다양한 사용자 요구를 수용

4. UI 설계 지침 (닐슨의 10가지 휴리스틱)

  1. 시스템 상태 가시성
  2. 시스템과 실세계의 일치
  3. 사용자 제어 및 자유
  4. 일관성과 표준
  5. 오류 예방
  6. 기억보다 인지
  7. 유연성과 효율
  8. 심플하고 명확한 디자인
  9. 오류 인지 및 복구 지원
  10. 도움말 및 문서화

5. UI 개발 단계

  1. 문제 정의: 사용자 조사, 요구사항 분석
  2. 개념 설계: 정보구조(IA), 내비게이션 설계
  3. 프로토타이핑: 와이어프레임 → 목업 → 프로토타입
  4. UI 구현: 실제 화면 개발
  5. 평가: 사용성 테스트, 휴리스틱 평가

6. 프로토타이핑 도구

도구 유형 설명
와이어프레임 (Wireframe) 화면 레이아웃을 간단한 선과 박스로 표현, 기획 단계
목업 (Mockup) 실제 디자인을 적용한 정적 화면
프로토타입 (Prototype) 인터랙션이 가능한 시뮬레이션 화면
스토리보드 (Storyboard) 화면 흐름과 시나리오를 순서대로 나열

7. UI 품질 요구사항

항목 설명
기능성 사용자 요구 기능 충족
신뢰성 오류 없이 안정적 동작
사용성 배우기 쉽고 사용하기 편리
효율성 최소한의 노력으로 목표 달성
유지보수성 변경·수정이 쉬움
이식성 다양한 환경에서 동작 가능

8. UX 설계 방법론

사용자 중심 설계 (UCD, User Centered Design)

  • 사용자의 필요와 특성을 최우선으로 설계

인지공학 (Cognitive Engineering)

  • 인간의 인지 특성을 고려한 설계

접근성 (Accessibility)

  • 장애인, 고령자 등 모든 사용자가 사용 가능하도록 설계
  • 웹 접근성 지침: WCAG (Web Content Accessibility Guidelines)
