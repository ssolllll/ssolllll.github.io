1. UML 개요

UML(Unified Modeling Language): 소프트웨어 시스템을 시각적으로 표현하기 위한 표준 모델링 언어.

  • 객체지향 분석·설계의 표준 표기법
  • 구조 다이어그램(정적) + 행위 다이어그램(동적)으로 구성

2. 구조 다이어그램 (Structure Diagram) - 정적 구조 표현

다이어그램 설명
클래스 다이어그램 클래스, 속성, 메서드, 클래스 간 관계 표현 (가장 많이 사용)
객체 다이어그램 특정 시점의 객체 인스턴스와 관계 표현
컴포넌트 다이어그램 소프트웨어 컴포넌트 구조와 의존 관계 표현
배치 다이어그램 하드웨어 노드와 소프트웨어 배치 관계 표현
패키지 다이어그램 요소들의 그룹화(패키지) 및 의존 관계 표현
복합구조 다이어그램 클래스 내부 구조와 협력 포트 표현

3. 행위 다이어그램 (Behavior Diagram) - 동적 행위 표현

다이어그램 설명
유스케이스 다이어그램 사용자(Actor)와 시스템 기능(UseCase) 간 관계 표현
순서 다이어그램 객체 간 메시지 교환을 시간 순서로 표현
통신 다이어그램 객체 간 상호작용을 네트워크 구조로 표현
상태 다이어그램 객체의 상태 변화 흐름 표현
활동 다이어그램 업무 흐름, 작업 순서, 분기를 표현 (순서도와 유사)
타이밍 다이어그램 시간에 따른 상태 변화 표현
상호작용 개요 다이어그램 활동 다이어그램 + 순서 다이어그램 결합

4. 클래스 다이어그램 - 관계 종류

관계 표기 설명
연관 (Association) 실선 화살표 두 클래스 간 관계 존재
의존 (Dependency) 점선 화살표 한 클래스가 다른 클래스를 일시적으로 사용
일반화 (Generalization) 실선 + 빈 삼각형 상속 관계 (IS-A)
실현 (Realization) 점선 + 빈 삼각형 인터페이스 구현 관계
집합 (Aggregation) 실선 + 빈 마름모 전체-부분 관계, 부분은 독립 존재 가능
복합 (Composition) 실선 + 채운 마름모 강한 전체-부분 관계, 부분은 독립 존재 불가

5. 유스케이스 다이어그램 구성 요소

요소 표기 설명
액터 (Actor) 사람 모양 시스템과 상호작용하는 외부 사용자/시스템
유스케이스 (UseCase) 타원 시스템이 제공하는 기능 단위
시스템 경계 사각형 시스템 범위
포함 (Include) 점선 <<include>> 반드시 포함되는 기능
확장 (Extend) 점선 <<extend>> 조건부로 추가되는 기능
일반화 실선 + 빈 삼각형 액터 또는 유스케이스 간 상속

6. 순서 다이어그램 구성 요소

  • 생명선 (Lifeline): 객체를 나타내는 수직 점선
  • 활성 박스 (Activation Box): 객체가 작업 중임을 나타내는 사각형
  • 메시지: 객체 간 통신 (동기, 비동기, 반환 메시지 구분)
  • 시간 흐름: 위에서 아래로

7. 스테레오타입 (Stereotype)

UML 요소에 의미를 추가하는 확장 표기법. << >> 기호 사용.

  • <<interface>>: 인터페이스
  • <<abstract>>: 추상 클래스
  • <<include>>, <<extend>>: 유스케이스 관계
  • <<boundary>>, <<control>>, <<entity>>: MVC 분류
