OpenClaw: 오픈소스 브라우저 에이전트의 가능성

Anthropic Computer Use를 넘어선 오픈소스 기반 웹 자동화 에이전트 구축기

📌 프로젝트 개요

특정 기업의 전유물이었던 ‘Computer Use’ 기능을 오픈소스 환경에서 구현하고자 OpenClaw를 도입했습니다. 단순한 웹 크롤링을 넘어, LLM이 직접 브라우저를 제어하고 복잡한 워크플로우를 자율적으로 수행하는 자율형 웹 에이전트를 구축하고 실무 적용 가능성을 검증한 프로젝트입니다.

  • 기술 스택: OpenClaw, Playwright, Claude 3.5 Sonnet / GPT-4o, Docker, FastAPI
  • 주요 기능: 시각적 요소 인식(SOM), 자율적 웹 네비게이션, 다단계 태스크 수행 및 오류 복구

💡 단계별 기술적 고도화 (Technical Evolution)

1. 프로프라이어터리 모델의 의존성 탈피

기존의 브라우저 제어 기능은 특정 모델(Anthropic 등)에 종속적인 경우가 많았습니다. OpenClaw를 통해 LLM-Agnostic(모델 불가지론)한 구조를 설계했습니다. 이를 통해 상황에 따라 GPT-4o나 로컬 LLM을 스위칭하며 비용과 성능의 균형을 맞출 수 있는 유연한 에이전트 환경을 조성했습니다.

2. 시각적 추론 기반의 요소 매핑 (SOM: Set of Marks)

텍스트 기반의 HTML 파싱만으로는 동적인 웹 페이지(SPA 등)를 제어하는 데 한계가 있었습니다.

  • Solution: OpenClaw의 시각적 강조 기능을 활용하여 브라우저 화면의 상호작용 가능한 요소들에 번호를 매기는 Set of Marks(SOM) 기법을 적용했습니다.
  • Effect: LLM이 좌표값 대신 시각적 인덱스를 바탕으로 클릭, 타이핑 등의 액션을 결정하게 함으로써, 복잡한 UI에서의 조작 정확도를 기존 방식 대비 약 40% 이상 향상시켰습니다.

3. Agentic Workflow: 자율적 오류 복구 및 피드백 루프

에이전트가 웹 서핑 중 예상치 못한 팝업이나 로딩 지연을 만났을 때 멈추는 문제를 해결하기 위해 Self-Correction 루프를 강화했습니다.

  • Feedback Loop: 액션 수행 후의 스크린샷을 찍어 이전 상태와 비교하고, 의도한 결과가 나오지 않았을 경우 스스로 다른 경로를 탐색하도록 프롬프트 체인을 고도화했습니다.
  • Integration: 이 과정을 MCP(Model Context Protocol)와 연결하여, 웹에서 수집한 정보를 즉시 사내 DB의 형식에 맞게 가공하여 저장하는 End-to-End 파이프라인을 구축했습니다.

📈 기대 효과 및 성과

  • 운영 비용 절감: 유료 솔루션 대안으로 오픈소스 프레임워크를 안정화하여 대규모 배치 작업 시 비용 최적화 달성
  • 워크플로우 확장성: 학자금 지원 서류 검증 등 기존의 정적인 OCR 작업을 넘어, 실시간 웹 데이터 검증이 필요한 영역으로 서비스 확장 가능성 확인
  • 기술적 자립도 확보: 오픈소스 커뮤니티의 최신 기술을 빠르게 내재화하여 팀 내 에이전트 기술력 상향 평준화

🛠️ 엔지니어의 노트: OpenClaw를 선택한 이유

기술 블로그를 운영하며 가장 중요하게 생각하는 것은 ‘투명성’입니다. OpenClaw는 내부 동작을 완전히 제어할 수 있어, 기업용 보안 환경에 맞게 커스텀하기에 최적의 선택지였습니다. 특히 브라우저 컨테이너화(Docker)를 통해 격리된 환경에서 에이전트를 구동함으로써 보안 이슈를 선제적으로 해결할 수 있었습니다.
