## 1. 소프트웨어 생명주기 (SDLC, Software Development Life Cycle)

소프트웨어를 개발하기 위한 과정을 단계별로 나눈 것.

**기본 단계**: 요구분석 → 설계 → 구현 → 테스트 → 유지보수

---

## 2. 소프트웨어 생명주기 모델

### 폭포수 모델 (Waterfall)
- 각 단계를 순차적으로 진행, 이전 단계로 돌아가기 어려움
- 요구사항이 명확할 때 적합
- 단계: 요구분석 → 설계 → 구현 → 테스트 → 유지보수

### 프로토타입 모델 (Prototype)
- 요구사항을 명확히 하기 위해 시제품(프로토타입)을 먼저 만들어 고객에게 확인
- 요구사항이 불명확할 때 적합
- 단점: 프로토타입 폐기 시 자원 낭비

### 나선형 모델 (Spiral)
- 폭포수 + 프로토타입 + 위험 분석 결합
- 반복적으로 진행하며 위험을 줄여나감
- 단계: 계획 → 위험분석 → 개발/검증 → 고객평가 (반복)
- 대형 프로젝트에 적합

### V 모델
- 폭포수 모델에 검증(테스트) 단계를 대칭 구조로 추가
- 각 개발 단계에 대응하는 테스트 단계 명확화

| 개발 단계 | 대응 테스트 단계 |
|-----------|----------------|
| 요구분석 | 인수 테스트 |
| 시스템 설계 | 시스템 테스트 |
| 상세 설계 | 통합 테스트 |
| 구현 | 단위 테스트 |

---

## 3. 애자일 방법론 (Agile)

변화에 유연하게 대응하는 반복·점진적 개발 방법론.

**핵심 가치 (애자일 선언문)**
- 개인과 상호작용 > 프로세스와 도구
- 작동하는 소프트웨어 > 포괄적인 문서
- 고객과의 협력 > 계약 협상
- 변화에 대응 > 계획 준수

### 스크럼 (Scrum)
- 스프린트(Sprint) 단위(1~4주)로 반복 개발
- 주요 역할: 제품 책임자(PO), 스크럼 마스터, 개발팀
- 주요 산출물: 제품 백로그, 스프린트 백로그, 번다운 차트
- 데일리 스크럼: 매일 15분 진행 상황 공유

### XP (eXtreme Programming)
- 고객 만족과 의사소통 강조, 짝 프로그래밍(Pair Programming) 특징
- 핵심 가치: 의사소통, 단순성, 피드백, 용기, 존중
- 주요 실천법

| 실천법 | 설명 |
|--------|------|
| 짝 프로그래밍 | 두 명이 한 컴퓨터에서 함께 개발 |
| TDD | 테스트 코드 먼저 작성 후 구현 |
| 지속적 통합 (CI) | 매일 여러 번 통합 및 빌드 |
| 리팩터링 | 기능 변경 없이 코드 구조 개선 |
| 소규모 릴리즈 | 작은 단위로 자주 배포 |

### 칸반 (Kanban)
- 시각적 보드(할 일 / 진행 중 / 완료)로 작업 흐름 관리
- WIP(Work In Progress) 제한으로 병목 방지

## 4. 소프트웨어 개발 방법론 종류

| 방법론 | 특징 |
|--------|------|
| 구조적 방법론 | 기능 중심, 프로세스와 데이터 분리, DFD 사용 |
| 정보공학 방법론 | 기업 전체 정보시스템을 계획적으로 구축 |
| 객체지향 방법론 | 객체 중심, UML 사용, 재사용성 높음 |
| 컴포넌트 기반(CBD) | 컴포넌트 재사용으로 개발 생산성 향상 |
| 애자일 방법론 | 반복·점진적, 변화에 유연 |

---

## 5. 소프트웨어 프로세스 성숙도 모델

### CMMI (Capability Maturity Model Integration)
소프트웨어 개발 조직의 프로세스 성숙도 평가 모델 (5단계)

| 레벨 | 명칭 | 설명 |
|------|------|------|
| 1 | 초기 (Initial) | 프로세스 정의 없음, 개인 역량에 의존 |
| 2 | 관리 (Managed) | 기본 프로젝트 관리 수행 |
| 3 | 정의 (Defined) | 조직 표준 프로세스 정의 |
| 4 | 정량적 관리 (Quantitatively Managed) | 정량적 측정·통제 |
| 5 | 최적화 (Optimizing) | 지속적 프로세스 개선 |

### SPICE (ISO/IEC 15504)
소프트웨어 프로세스 평가 국제 표준 (6단계: 0~5)

## 6. 비용 산정 기법

### 하향식 산정
- **델파이 기법**: 전문가 의견을 익명으로 수렴·반복하여 합의 도출

### 상향식 산정
- **LOC (Lines of Code)**: 코드 라인 수 기준 산정
- **개발 단계별 산정**: 단계별 노력 합산

### 수학적 산정 모델
| 모델 | 특징 |
|------|------|
| COCOMO | LOC 기반, 유기형/반분리형/내장형 구분 |
| 기능 점수(FP) | 기능 단위로 규모 측정, 언어 독립적 |
| Putnam | 시간에 따른 인력 투입 분포 모델 |
