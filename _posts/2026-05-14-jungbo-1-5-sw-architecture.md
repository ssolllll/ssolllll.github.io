## 1. 소프트웨어 아키텍처 개요

소프트웨어의 전체적인 구조를 설계하는 것. 컴포넌트, 컴포넌트 간 관계, 설계 원칙 등을 정의.

**아키텍처 설계 목표**: 성능, 보안, 가용성, 유지보수성, 확장성 등 품질 속성 달성

---

## 2. 아키텍처 패턴 종류

### 레이어드 패턴 (Layered Pattern)
- 시스템을 계층(Layer)으로 구분
- 각 계층은 하위 계층에만 의존
- 예: OSI 7계층, MVC

### 클라이언트-서버 패턴 (Client-Server)
- 서버가 서비스 제공, 클라이언트가 서비스 요청
- 예: 웹 서비스

### MVC 패턴 (Model-View-Controller)
- Model: 비즈니스 로직, 데이터
- View: 사용자 인터페이스
- Controller: Model과 View 연결, 입력 처리
- 관심사 분리로 유지보수성 향상

### 파이프-필터 패턴 (Pipe-Filter)
- 데이터 스트림을 필터(처리 단계)와 파이프(연결)로 처리
- 예: Unix 쉘 파이프(`|`), 컴파일러

### 브로커 패턴 (Broker)
- 분산 환경에서 컴포넌트 간 통신을 중개자(Broker)가 담당
- 예: Apache Kafka, RabbitMQ

### 이벤트 버스 패턴 (Event-Bus)
- 이벤트 소스, 이벤트 리스너, 채널, 버스로 구성
- 느슨한 결합 달성

### 마이크로서비스 아키텍처 (MSA)
- 애플리케이션을 작은 독립 서비스 단위로 분리
- 각 서비스는 독립 배포, 독립 확장 가능

---

## 3. 모듈화 (Modularity)

시스템을 독립적인 모듈 단위로 분리하는 것.

**모듈화의 장점**
- 재사용성 향상
- 유지보수 용이
- 복잡성 감소
- 독립적 개발 가능

---

## 4. 결합도 (Coupling)

모듈 간 상호 의존성의 정도. **낮을수록 좋음**.

| 결합도 유형 | 설명 | 수준 |
|-------------|------|------|
| 내용 결합 (Content) | 한 모듈이 다른 모듈 내부를 직접 참조 | 가장 나쁨 |
| 공통 결합 (Common) | 공유 데이터(전역변수)를 함께 사용 | |
| 외부 결합 (External) | 외부 형식, 프로토콜 공유 | |
| 제어 결합 (Control) | 제어 플래그를 전달하여 흐름 제어 | |
| 스탬프 결합 (Stamp) | 자료구조를 매개변수로 전달 | |
| 자료 결합 (Data) | 단순 데이터만 매개변수로 전달 | 가장 좋음 |

> 암기: **내공외제스자** (내용-공통-외부-제어-스탬프-자료)

---

## 5. 응집도 (Cohesion)

모듈 내부 요소들이 얼마나 밀접하게 연관되어 있는지. **높을수록 좋음**.

| 응집도 유형 | 설명 | 수준 |
|-------------|------|------|
| 우연적 응집 (Coincidental) | 관련 없는 요소들이 단순히 같이 묶임 | 가장 나쁨 |
| 논리적 응집 (Logical) | 유사한 종류의 기능들이 묶임 |  |
| 시간적 응집 (Temporal) | 같은 시간에 실행되는 요소들이 묶임 | |
| 절차적 응집 (Procedural) | 순서가 있는 작업들이 묶임 | |
| 교환적 응집 (Communicational) | 같은 데이터를 사용하는 기능들이 묶임 | |
| 순차적 응집 (Sequential) | 출력이 다음 입력으로 사용되는 요소들 묶임 | |
| 기능적 응집 (Functional) | 하나의 기능만 수행 | 가장 좋음 |

> 암기: **우논시절교순기** (우연-논리-시간-절차-교환-순차-기능)

---

## 6. 팬인(Fan-In) / 팬아웃(Fan-Out)

| 용어 | 설명 |
|------|------|
| 팬인 (Fan-In) | 해당 모듈을 호출하는 모듈의 수. 높을수록 재사용성 높음 |
| 팬아웃 (Fan-Out) | 해당 모듈이 호출하는 모듈의 수. 낮을수록 좋음 |

## 7. 소프트웨어 품질 속성 (ISO/IEC 25010)

| 품질 특성 | 설명 |
|-----------|------|
| 기능 적합성 | 기능이 요구사항을 충족하는 정도 |
| 성능 효율성 | 자원 사용 대비 성능 |
| 호환성 | 다른 시스템과 공존·연동 가능 여부 |
| 사용성 | 사용자가 쉽게 사용할 수 있는 정도 |
| 신뢰성 | 정해진 조건에서 기능을 수행하는 정도 |
| 보안성 | 정보 보호 수준 |
| 유지보수성 | 수정·개선의 용이함 |
| 이식성 | 다른 환경으로 이전 가능 여부 |
