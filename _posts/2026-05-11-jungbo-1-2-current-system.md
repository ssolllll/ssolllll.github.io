## 1. 현행 시스템 파악 개요

새로운 시스템 개발 전 현재 운영 중인 시스템의 구성과 기능을 분석하는 작업.
목적: 개발 범위 및 이행 방향 설정, 현행 시스템의 문제점 파악.

---

## 2. 현행 시스템 파악 절차

1단계: 시스템 구성 파악 → 2단계: 시스템 기능 파악 → 3단계: 시스템 인터페이스 파악
4단계: 아키텍처 파악 → 5단계: 소프트웨어 구성 파악
6단계: 하드웨어 구성 파악 → 7단계: 네트워크 구성 파악

---

## 3. 파악 대상 구성 요소

### 소프트웨어 구성 파악
- 운영체제(OS), DBMS, 미들웨어, 오픈소스 등 파악
- 버전, 라이선스 정책 확인 필수

### 하드웨어 구성 파악
- 서버(웹, WAS, DB, 파일), 클라이언트 장비 목록
- 이중화(HA, High Availability) 여부 확인

### 네트워크 구성 파악
- 네트워크 구성도 작성
- 방화벽, L4/L7 스위치, DMZ 구성 파악
- 구간별 통신 프로토콜 확인

---

## 4. 소프트웨어 아키텍처 파악

| 계층 | 설명 |
|------|------|
| 클라이언트 계층 | 사용자 인터페이스 (웹 브라우저, 앱) |
| 웹 서버 계층 | 정적 콘텐츠 처리, Apache/Nginx |
| WAS 계층 | 동적 처리, 비즈니스 로직 (Tomcat, JBoss) |
| DB 계층 | 데이터 저장 및 관리 (Oracle, MySQL 등) |

---

## 5. 현행 시스템 문서화 산출물

| 산출물 | 설명 |
|--------|------|
| 시스템 구성도 | 전체 시스템 구성 요소 및 관계 도식화 |
| 네트워크 구성도 | 네트워크 장비 및 연결 구조 도식화 |
| 소프트웨어 구성 목록 | 사용 중인 SW, 버전, 라이선스 목록 |
| 하드웨어 구성 목록 | 서버 사양, 수량, 설치 위치 목록 |

---

## 6. 개발 기술 환경 파악 시 고려 사항

- **운영체제**: 신규 개발 시스템과의 호환성 검토
- **DBMS**: 데이터 마이그레이션 가능 여부
- **미들웨어**: 연동 인터페이스 방식(API, MQ, ESB 등) 확인
- **오픈소스**: 라이선스 유형(GPL, MIT, Apache 등) 검토 필수
